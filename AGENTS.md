# AGENTS.md

## 第一性原則

### 一律使用中文與用戶互動
### 隨時更新/建立會話遷移指南(專案根目錄/session_migration.md)，把工作進度、todo、下一步將會先做什麼包含細節清楚寫入，以免突遇上下文壓縮遺失細節
### 修bug時主動檢查相關代碼，非本次直接的bug要整理紀錄(session_migration.md)下來
### 每個階段修改後要進行型別檢查和lint驗證
### 新功能必須有對應的測試
### 斷點保護機制:在執行高風險操作(如大量刪除、修改核心Auth邏輯)前，必須主動提示用戶進行Git Commit
### 大改動前先輸出「修改計畫」，待確認後再執行。除非用戶明確的說直接開始執行
